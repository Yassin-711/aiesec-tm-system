## ðŸ“‹ Common Formulas Reference

### People Tab Formulas

**Column A - Member ID (Row 2):**
```
="TM"&TEXT(ROW()-1,"0000")
```

**Column E - Team Leader (Row 2):**
```
=IF(C2="Team Leader","N/A",IF(C2="VP TM","N/A",IF(D2="TM","N/A",FILTER(People!B:B,People!C:C="Team Leader",People!D:D=D2))))
```

**Column J - Attendance % (Row 2):**
```
=IFERROR(COUNTIF(Attendance!B:B,A2)/COUNTIF(Meetings!A:A,"<>")*100,0)
```

**Column K - Pulse Score (Row 2):**
```
=IFERROR(AVERAGE(FILTER(Weekly Pulse!J:J,Weekly Pulse!B:B=A2,Weekly Pulse!A:A>=TODAY()-28)),0)
```

**Column L - Task Score (Row 2):**
```
=IFERROR(SUMIF(Tasks!B:B,A2,Tasks!H:H)/MAX(COUNTIF(Tasks!B:B,A2),1),0)
```

**Column M - Engagement Status (Row 2):**
```
=IF(OR(J2<60,K2<2),"RED",IF(OR(J2<80,K2<3),"YELLOW","GREEN"))
```

### Main Functions

| Function | Purpose | When to Use |
|----------|---------|-------------|
| `logAttendance(memberId, meetingId)` | Log attendance from QR scan | Called from attendance form |

| `generateMonthlyRecognition()` | Generate monthly recognition | Monthly trigger (1st) |

| `getTeamData(teamLeaderName)` | Get team data for TL dashboard | API call from web interface |
| `getVPTMDashboardData()` | Get VP TM dashboard data | API call from web interface |
| `createMeeting(...)` | Create new meeting | Called from TL dashboard |
| `assignTask(...)` | Assign task to member | Called from TL dashboard |

| Function | Purpose |

## ðŸŽ¯ Common Tasks

### Adding a New Member

1. Go to **People** tab
2. Add new row with:
   - Name
   - Role Type: "Member"
   - Function: Select from dropdown
   - Team Leader: Select from dropdown (auto-filtered by function)
   - Email, Phone, Start Date
   - Status: "Active"
3. Member ID auto-generates
4. Formulas auto-calculate metrics

### Creating a Meeting

1. Go to **Meetings** tab
2. Add new row with:
   - Meeting Name
   - Function
   - Team Leader
   - Date, Time, Location
3. Meeting ID auto-generates
4. QR Code Link and Form Link generated by Apps Script

### Assigning a Task

1. Go to **Tasks** tab
2. Add new row with:
   - Assigned To: Select member
   - Assigned By: Select Team Leader/VP TM
   - Function: Auto-filled
   - Task Name
   - Due Date
   - Status: "Not Started"
3. Task ID auto-generates
4. Performance Score auto-calculates on completion

### Checking At-Risk Members

1. Go to **VP TM Dashboard**
2. View "At-Risk Members" section
3. Or filter **People** tab by Engagement Status = "RED"
4. Alerts automatically sent to TL and VP TM

### Generating Recognition

1. Run `generateMonthlyRecognition()` function manually
2. Or wait for monthly trigger (1st of month)
3. Check **Recognition** tab
4. Email sent to all active members

## ðŸ” Troubleshooting Quick Fixes

### Formula Shows #REF! or #N/A
- **Fix**: Check if referenced sheet/tab exists
- **Fix**: Verify column references are correct
- **Fix**: Check if data exists in referenced range

### Attendance % Shows 0%
- **Fix**: Verify member has attendance records
- **Fix**: Check if meeting exists
- **Fix**: Verify formula references correct columns

### Pulse Score Not Updating
- **Fix**: Check if pulse entries exist
- **Fix**: Verify date range in formula (last 28 days)
- **Fix**: Check if Member ID matches

### Apps Script Not Working
- **Fix**: Check execution logs (View > Execution log)
- **Fix**: Verify permissions are granted
- **Fix**: Test function manually
- **Fix**: Check quota limits

### Web Interface Not Loading
- **Fix**: Verify Apps Script URL is correct
- **Fix**: Check deployment is active
- **Fix**: Clear browser cache
- **Fix**: Try different browser

### Emails Not Sending
- **Fix**: Check spam folder
- **Fix**: Verify email addresses are correct
- **Fix**: Check Apps Script quotas
- **Fix**: Verify triggers are set up

## ðŸ“± Mobile Access

### QR Code Attendance
1. Member scans QR code with phone
2. Opens attendance form
3. Enters Member ID (or pre-filled)

5. Clicks "Check In"

## ðŸ”„ Cycle Management

### Starting New Cycle

1. **Archive Old Data**
   - Create new sheet: "Cycle [Date] Archive"
   - Copy old data
   - Keep for reporting

2. **Update Member Statuses**
   - Set inactive members to "On Hold" or "Left"
   - Add new members as "Active"
   - Update Team Leader assignments

3. **Reset Metrics (Optional)**
   - Clear old attendance (or archive)
   - Clear old pulse entries (or archive)
   - Reset task scores

4. **Update Structure**
   - Verify Team Leaders
   - Update function assignments
   - Create new meetings

### Ending Cycle

1. **Finalize Data**
   - Complete all tasks
   - Final pulse check
   - Calculate final metrics

2. **Generate Reports**
   - Cycle performance report
   - Recognition list
   - Member retention stats

3. **Archive**
   - Export to Excel backup
   - Archive to separate sheet
   - Prepare for next cycle
